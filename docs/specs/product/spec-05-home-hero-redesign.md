> 트랙: [Design] 실배포 전 리디자인 — 리포트 B 시리즈 화면 재설계 (B-1, P0). 정본 진행상태 = `docs/roadmap.md` + `design-redesign-handoff.md`

## 1. 배경 · 문제

홈(HomeView)은 방문자의 첫 화면이자 가입 전환의 진입점이다. 현재 구현은 **가치 제안 없이 기능 카드 5개를 평면 나열**해 "이게 나에게 왜 좋은가"를 못 전달한다. 컨설팅 리포트(`docs/consulting/out/design-ux-report.md` 총평 §29, A-1 §46~48, 목표형태 B-1 §140~159, 로드맵 P0 §228 #4) 결손:

- **P0 · 가치 제안 부재.** 히어로·헤드라인·서브카피 없이 곧장 ①선수지식 ②진단 다운로드 ③채점 ④AI결과 ⑤맞춤출제 기능 카드 평면 나열 → 방문자가 "왜 좋은가"를 못 얻고 이탈.
- **설명이 툴팁에 숨어 있음.** 각 카드 기능 설명이 `v-tooltip`(호버) 안에 있어 모바일·첫 방문자에게 안 보임.
- **위생.** `<script>`·`<template>` 상단에 죽은 주석 코드(get/post 테스트, 주석 처리된 회원가입 안내) 다수. 카드 라벨 `text-2xl/4xl` 남발(리포트 P1 #7과 연결되나 토큰 전역정리는 Out).

### 1.1 사실확인 (코드 레벨)

- 현재 `HomeView.vue`는 **데이터 호출 없음** — `conceptImageUrl` computed 하나로 `images/concept-for-home.png` 정적 이미지만 렌더. API/스토어 의존 0.
- CTA 목적지 라우트(존재 확인 `web/src/router/index.js`): `/diagnosis`(DiagView) · `/concept`(ConceptView) · `/record` · `/result`(ResultView, 비로그인 샘플) · `/personal`.
- 상단 글로벌 내비(셸 전환)는 **P1 별도 트랙**(리포트 §234 #5, 열린질문 §248 #2) → 본 spec 범위 밖. 현 `AppLayout`(사이드바) 유지.

## 2. 범위

### In
1. **히어로 블록** — H1 헤드라인 + 서브카피 + 1차 CTA 2개. 톤 = **학부모(신뢰 톤)**(사용자 결정 2026-06-24).
   - CTA-1(주): "무료로 진단 시작" → `/diagnosis`.
   - CTA-2(보조): "개념 그래프 둘러보기 (로그인 불필요)" → `/concept`.
2. **작동방식 4스텝** — 좌→우 인과 흐름: ①진단지 받기 → ②채점 → ③AI 약점 분석 → ④맞춤 학습지. 툴팁에 숨겼던 설명을 본문 한 줄로 노출.
3. **기능 카드 재정리** — 기존 5카드를 4스텝과 중복 제거하며 정리. 그래프 미리보기(`concept-for-home.png`)는 보조 영역으로 보존하고 "샘플 결과 보기 →"(`/result`) 보조 링크 추가.
4. **위생 정리** — 죽은 주석 코드 제거, 호버 전용 설명 본문화, 라벨 타이포 과용 완화(히어로/스텝 한정, 전역 토큰화는 Out).

### Out (이번 범위 밖)
- **상단 글로벌 내비/셸 전환** — P1 별도 트랙. `AppLayout` 사이드바 유지.
- **디자인 토큰 1벌 + 타이포 스케일 전역 정리(C안, P1 #7)** — Later. 본 spec은 HomeView 내부 클래스만 손댐.
- **사회적 증거(후기/사용자 수) 실데이터** — 자리(placeholder)만 두지 않음. 허위 지표 노출 금지 → 실데이터 확보 후 후속.
- **게이미피케이션/배지·스트릭** — 리포트 P2.
- **백엔드·라우터·스토어 변경** — 없음. 순수 뷰 표현.
- **신규 일러스트/이미지 제작** — 기존 `concept-for-home.png` 재사용.

## 3. 결정 (사용자 승인 2026-06-24)

- **D1 · 톤 = 학부모(신뢰 톤).** 헤드라인·서브카피를 신뢰·효용 중심 존댓말로. "진단 한 번으로 약한 고리를 찾아 바로 채울 학습지를 받아보세요" 류. (학생 동기부여 톤·게이미피케이션은 후속 옵션, 리포트 열린질문 #1.)
- **D2 · 범위 = 풀 B-1(히어로 + 4스텝 + 카드 재정리) 한 패스.** 단일 정적 뷰·완전 가역·리스크 승수 없음 → 분리 이득 없음(사용자가 판단 위임, 어시스턴트 결정).
- **D3 · 셸 내비 제외, 사이드바 유지.** 셸 전환은 손 많이 가는 P1 별도 의사결정 → 본 spec과 분리.
- **D4 · 사회적 증거는 허위 지표 금지.** 실데이터 없으면 노출 안 함.

## 4. 설계

### 4.1 레이아웃 (PrimeFlex grid, 사이드바 콘텐츠 영역 안)
```
┌ 히어로 (col-12, 카드/배경 강조) ───────────────────────────────┐
│  H1: "수학은 계단입니다. 어디서 막혔는지 AI가 짚어드립니다."     │
│  서브: 이전 개념이 다음 학습을 좌우합니다. 진단 한 번으로         │
│        약한 고리를 찾아 바로 채울 학습지를 받아보세요.            │
│  [ 무료로 진단 시작 ](주, /diagnosis)                            │
│  [ 개념 그래프 둘러보기 · 로그인 불필요 ](보조, /concept)        │
└────────────────────────────────────────────────────────────────┘
┌ 작동 방식 (col-12, 4스텝 좌→우, 화살표/번호) ──────────────────┐
│  ①진단지 받기 → ②채점하기 → ③AI 약점 분석 → ④맞춤 학습지      │
│  (각 스텝: 아이콘 + 라벨 + 한 줄 설명. 해당 기능 라우트로 링크)  │
└────────────────────────────────────────────────────────────────┘
┌ 그래프 미리보기 + 샘플 (col-12) ───────────────────────────────┐
│  [concept-for-home.png]   "샘플 결과 보기 →" (/result)          │
│  "선수 지식 알아보기" → /concept                                │
└────────────────────────────────────────────────────────────────┘
```

### 4.2 4스텝 ↔ 기존 카드 매핑 (중복 제거)
| 스텝 | 라벨 | 한 줄 설명(툴팁 본문화) | 라우트 |
|---|---|---|---|
| ① | 진단 학습지 받기 | 중단원별 시험지로 개념 이해 여부 확인 | `/diagnosis` |
| ② | 채점하기 | 채점 결과를 AI에게 전달 | `/record` |
| ③ | AI 약점 분석 | 어떤 개념이 부족한지·필요한 선수지식 확인 | `/result` |
| ④ | 맞춤 학습지 출제 | 분석에 따라 나에게 딱 맞는 문제 출제 | `/personal` |

기존 "①선수지식 알아보기" 카드는 4스텝(진단 흐름)과 성격이 달라 **히어로 CTA-2 + 그래프 미리보기 영역**으로 흡수. 5카드 평면나열은 폐기, 정보는 히어로/스텝/미리보기로 재배치(기능 손실 0, 모든 라우트 진입점 보존).

### 4.3 카피 (학부모 신뢰 톤, 초안 — 코드 단계에서 확정)
- H1: **"수학은 계단입니다. 어디서 막혔는지 AI가 짚어드립니다."**
- 서브: "이전 개념의 이해가 다음 학습을 좌우합니다. 진단 한 번으로 약한 고리를 찾아, 바로 채울 맞춤 학습지를 받아보세요."
- CTA-1: "무료로 진단 시작" / CTA-2: "개념 그래프 둘러보기 (로그인 불필요)"

### 4.4 스타일
- PrimeFlex/PrimeVue 기존 유틸리티 클래스 사용(Tailwind 미도입). 히어로 CTA는 PrimeVue `Button`(주=primary, 보조=outlined/secondary)로 통일 — 현 `router-link`+`card` 클릭영역 대비 버튼 affordance 명확화.
- 타이포: 히어로 H1만 큰 스케일(예 `text-4xl`/`text-5xl`), 서브·스텝은 절제. 무의미한 `text-2xl` 라벨 남발 제거.
- 반응형: 4스텝은 `col-12 md:col-3`(모바일 세로 스택 → 데스크톱 가로 4열).
