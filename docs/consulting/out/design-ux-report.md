# MMT 디자인/UX 컨설팅 리포트

> 작성: 시니어 프로덕트 디자이너 관점 컨설팅 (프롬프트 01)
> 분석 대상: `web/src/views/` 10개 · `web/src/layout/` 셸 · `assets/styles.scss` · 테마 설정
> 분석 일자 기준 코드 스냅샷. **코드는 수정하지 않음 (컨설팅 산출물).**
> 스택 확인: Vue 3 + PrimeVue 3.39 **Sakai 관리자 템플릿** + PrimeFlex, 테마 `lara-light-indigo`(인디고 프라이머리), 그래프 Cytoscape+klay, 표 PrimeVue DataTable, PDF html2pdf.

## 0. 사실 확인 / 브리프와 실제 코드의 차이

분석에 들어가기 전, 브리프 가정과 실제 코드가 다른 지점을 먼저 못 박는다. (추측 아님 — 코드 확인)

- **ResultView에는 차트가 없다.** 브리프는 "Chart.js 차트 + 리포트"라 했으나, `ResultView.vue`는 **PrimeVue DataTable(시급도 상/중/하 뱃지) + Cytoscape 선수지식 TREE**로 구성된다. `chart.js@3.3.2`는 의존성에 있고 `Chart`가 `main.js`에 전역 등록되어 있지만 **어떤 `.vue`에서도 `<Chart>`를 쓰지 않는다.** → 차트 비평이 아니라 "표를 어떻게 사람이 읽을 보고서로 바꿀까"가 진짜 과제다.
- **셸은 PrimeVue Sakai 관리자 템플릿 그대로다.** `AppLayout`(layout-sidebar + layout-topbar + layout-main) + `AppMenu`(그룹형 사이드바 메뉴)는 백오피스/대시보드용 골격이다. → "백오피스 인상" 가설은 **사실로 확인됨**(아래 A-0).
- **미완성/잠재 버그 흔적이 사용자에게 노출된다.** PersonalView의 "개발 중" 회색 오버레이(런타임에 보임), `PersonalView` 템플릿의 `isSet` 미정의 참조, `DiagView`·`ResultView`에 복붙된 로그인 다이얼로그가 그 파일에 없는 `login`/`loginErrorMessage`를 참조, `ErrorView`가 import는 주석 처리된 `<AppConfig simple />`를 렌더 시도 — 이들은 02번(구현) 영역이지만 **디자인 인상에 직접 영향**을 주므로 P 태그로 함께 표기한다.

## 1. 요약 — 가장 중요한 3가지

1. **"관리자 템플릿"이 제품의 정체성을 누른다 (P0).**
   사이드바+토프바 백오피스 셸, 영문/한글 혼용 제목("School Level", "Grade Level", "Representative"), 모든 화면이 동일한 `card` 격자 — B2C 학습 서비스가 아니라 **사내 어드민**처럼 보인다. 가장 적은 노력으로 가장 큰 인상 개선이 나오는 지점도 여기다(셸·카피·컬러).

2. **핵심 두 화면이 "데이터는 있는데 의미가 없다" (P0).**
   ConceptView(그래프)는 처음 본 학생에게 **압도적이고 길 안내가 없다**(7px 초소형 노드, 호버 전용 하이라이트, 줌/검색/현재위치 없음, 노란색 노드는 흰 배경에서 안 보임). ResultView(AI 결과)는 **밀도 높은 표 + 상/중/하 뱃지**를 설명 없이 던져 "그래서 뭘 해야 하지"로 이어지지 않는다. 이 두 화면이 제품의 차별점인데 지금은 차별점을 못 전달한다.

3. **미완성·신뢰 신호 결손이 라이브에서 그대로 보인다 (P0~P1).**
   PersonalView의 "개발 중" 오버레이, 영문 토스트("Confirmed"), 사과형 푸터("화면 멈춤 및 오류 발생 시 개발자에게 문의"), 가치 제안 없는 홈 — 방문자에게 **"미완성 사이드 프로젝트"** 신호를 준다. 신뢰(가입 전환)에 직접 타격.

> **한 줄 처방:** 컴포넌트를 다시 그리기 전에 **(a) 셸을 학습 서비스 구조로, (b) 카피를 한국어·학생 언어로, (c) 컬러·타이포 토큰을 하나로** 통일하는 것만으로 "2년 전 템플릿" 인상의 70%가 사라진다. 리디자인(B)·디자인 시스템(C)은 그 위에 얹는다.

## 2. A. 휴리스틱 진단 (화면별 · 심각도 · 근거)

심각도: **P0**=배포 전 반드시 / **P1**=곧 / **P2**=나중. 근거는 [휴리스틱 / 사용자 영향]으로 표기.

### A-0. 공통 셸 (AppLayout / AppTopbar / AppSidebar / AppMenu / AppFooter)

- **P0 · 백오피스 셸이 B2C 학습 서비스와 불일치.** 좌측 그룹형 사이드바(Home/선수 지식/맞춤 학습/마이 페이지) + 상단 토프바는 Sakai 어드민 골격. [심미성·최소주의(Nielsen #8) / 학생·학부모가 "학습 도구"가 아니라 "관리 콘솔"로 인지 → 첫인상 신뢰 하락]
- **P1 · 토프바 인증이 빈약.** 우상단에 사용자 아이콘 하나(`pi pi-user`) → 클릭하면 로그인 다이얼로그. 로그인 상태는 "로그/아웃"이 **두 줄로 줄바꿈**된 텍스트 링크. [인지 일치 / 로그인 여부·내 정보가 한눈에 안 보임, 줄바꿈 텍스트는 미완성 인상]
- **P1 · 푸터 카피가 사과·디버그 톤.** "화면 멈춤 및 오류 발생 시 개발자에게 문의주세요." [신뢰 신호 / 제품이 불안정하다고 먼저 선언하는 꼴]
- **P2 · 사이드바 메뉴 뱃지 'NEW'가 '회원 정보 수정'에 붙어 있음.** [정보 위계 / 핵심 가치가 아닌 곳에 강조 낭비]

### A-1. HomeView (랜딩/홈)

- **P0 · 가치 제안 부재.** 히어로·헤드라인·서브카피 없이 곧장 ①선수지식 ②진단 다운로드 ③채점 ④AI결과 ⑤맞춤출제 **기능 카드 평면 나열**. [정보 위계 / 방문자가 "이게 나에게 왜 좋은가"를 못 얻고 이탈]
- **P1 · 플로우가 카드에 숨어 있다.** ①~⑤는 실제로 **순차 워크플로우**(진단→채점→분석→맞춤)인데 동급 카드라 순서·인과가 안 읽힘. 번호는 `&#9312;` 같은 원문자 HTML 엔티티. [인지 일치 / 첫 사용자가 어디서 시작할지 모름]
- **P1 · 설명이 툴팁(`v-tooltip`)에 갇힘 + 줄바꿈 미작동.** 카드 의미를 `\n` 포함 문자열 툴팁으로 전달하는데 일반 텍스트 툴팁에서 `\n`은 줄바꿈으로 렌더되지 않음(모바일엔 호버도 없음). [발견 가능성 / 핵심 설명을 못 봄]
- **P2 · "AI 분석 결과보기"만 빨간 테두리 카드 + "(클릭해서 샘플을 확인하세요.)".** 의도는 좋으나 빨강은 보통 에러/경고 신호. [색 일관성]

### A-2. ConceptView (개념 지식 그래프 — 핵심 ★ 깊게)

- **P0 · 첫 진입에 그래프가 비어 있고, 도달까지 4단계.** School Level→Grade Level→대/중/소단원 Tree→개념 목록→[선수지식 확인] 버튼을 다 거쳐야 그래프가 그려짐. [사용자 제어·인지 부하 / "서비스의 얼굴"이 첫 화면에선 텅 빈 카드 4개]
- **P0 · 그래프 가독성·접근성 결손.**
  - 노드 7px·폰트 7px로 **극단적으로 작다**(`nodeSize=7, fontSize=7`). [가독성]
  - 색 범례가 CSS 색 이름 raw 값: `yellow`, `springgreen`, `lightpink`… **노란색 노드는 흰 카드 배경에서 거의 안 보임**(대비 실패). [접근성 대비 — WCAG 1.4.11 / 색맹·저시력 사용자 구분 불가, 12색 학년 구분도 과다]
  - 하이라이트가 **호버 전용**(`mouseover`)이고 `mouseout`에 리셋 → **선택 상태가 유지되지 않음**. 모바일/태블릿은 호버가 없어 핵심 인터랙션이 사실상 불가. [접근성·터치 타깃 / 모바일 사용자 핵심 기능 상실]
  - 줌 컨트롤·미니맵·검색·"현재 보고 있는 개념" 표식 없음. 노드 많으면 길 잃음. [사용자 제어·가시성]
- **P1 · 영문 어드민 제목.** "School Level", "Grade Level"이 학생 화면에 영문으로. [인지 일치·일관성]
- **P1 · 안내가 도움말 아이콘 뒤에 숨음.** 조작법(스크롤=확대, 점 클릭=상세)이 `?` OverlayPanel 안에만 있음. [발견 가능성]
- **P2 · 빈 상태가 "라벨만 있는 빈 표".** "개념을 선택해주세요" + 값이 빈 4행 리스트. [빈 상태 디자인 / 안내가 아니라 깨진 표처럼 보임]

### A-3. ResultView (AI 분석 결과 — 핵심 ★ 깊게)

- **P0 · 핵심 보상이 "raw 표"로 던져진다.** 시급도 상/중/하 뱃지가 붙은 DataTable + 누적 Cytoscape 트리. **요약 서사·점수·"가장 급한 약점 N개" 같은 헤드라인이 없음.** 학생/학부모가 한눈에 "내 약점은 X, 그래서 Y를 해라"로 못 감. [인지 일치·정보 위계 / "그래서 뭘 하지"가 비어 있음 — 제품 핵심 가치 미전달]
- **P0 · 영문/내부용어 노출.** 컬럼 헤더 `Representative`가 화면에 영문 그대로, "선수지식 깊이", "toConceptDepth" 같은 내부 모델 용어가 사용자에게 노출. [인지 일치 / 비전문가 이해 불가]
- **P1 · "시급도 상/중/하"의 산정이 불투명.** 확률 백분위를 3등분해 상/중/하를 매기는데(코드 `setPriority`) 사용자에겐 근거가 안 보임. [신뢰·설명가능성 / AI 결과 신뢰 하락]
- **P1 · 뱃지 색 의미 충돌.** 상=danger(빨강)/중=warning(주황)/'new'=success(초록). 'new'가 코드에 남아 있고 색 의미 체계가 임시. [색 일관성]
- **P2 · 그래프 영역은 ConceptView와 동일한 접근성 문제 반복**(7px 노드, 호버 전용, 대비). [재사용된 문제]

### A-4. DiagView (진단 학습지 다운로드)

- **P1 · 첫 화면 상단이 빨간 경고 카피로 시작.** "저작권 제약으로 실제 문제가 제공되지는 않습니다." [신뢰·기대 관리 / 필요한 고지지만 빨강+상단 배치는 "쓸 수 없는 서비스" 인상 — 톤 다운 필요]
- **P1 · 4영역(레벨/학년/목록/미리보기) 동시 노출로 밀도 과다.** [인지 부하]
- **P2 · 버튼 라벨이 곧 상태 메시지.** "학습지를 선택하세요.", "로그인을 해주세요." 등 disabled 버튼 라벨로 안내. 기능적이나 구식 패턴. [표준 일치 / 토스트·헬퍼텍스트로 분리 권장]
- **P2 · 다운로드 다이얼로그 카피에 빨강 다용.** [색 일관성]

### A-5. RecordView (채점하기)

- **P1 · 정오답 입력이 ToggleButton o/x.** 한 문항씩 토글. 빠른 일괄 채점 UX(전체 O, 빠른 키보드 입력) 부재. [효율성]
- **P2 · "이전" 버튼이 홈으로 가지만 라벨은 "이전".** 일부 화면은 `router.go(-1)`, 일부는 홈으로 — **"이전"의 의미가 화면마다 다름**. [일관성]
- **P2 · 미리보기 빈 문항이 더미 이미지(empty001.jpg)·"~에 대한 문항입니다".** [빈 상태 / 임시 자산 노출]

### A-6. PersonalView (맞춤 학습지 출제)

- **P0 · "개발 중" 회색 오버레이가 라이브에 노출.** 조건 패널 위에 `rgba(0,0,0,0.3)` 박스 + "새로운 기능이 곧 추가됩니다!"가 런타임에 보임. [신뢰·완성도 / 미완성 제품 인상 — 가장 강한 "2년 전/공사 중" 신호]
- **P0 · 미완성 분기.** 템플릿이 미정의 `isSet`을 참조(`v-else-if="!isSet"`)해 버튼 분기가 깨질 수 있음. [구현 위생 → 02번 / 디자인 인상엔 직접 영향]
- **P1 · "출제하기"가 항상 disabled.** 조건 UI는 있으나 기능이 막혀 있어 화면 목적이 무력. [기능 일치]

### A-7. SignUpView (회원가입)

- **P1 · 영문 필드 라벨 + 긴 단일 컬럼 폼.** ID/Password/Password Confirm/Name/BirthDate/Comments 전부 영문 라벨에 매우 큰 타이포(text-2xl). [인지 일치·정보 위계 / 모바일에서 한없이 길어짐]
- **P1 · 검증 메시지를 disabled 버튼 라벨로 표현.** "[중복확인]을 해주세요.", "[비밀번호]가 조건을 만족하지 않습니다." 오타("울 해주세요")도 있음. [표준 일치·완성도 / 인라인 필드 검증이 표준]
- **P2 · "BirthDate: 연도를 클릭하여…" 안내.** Calendar 컴포넌트 사용성 보완 멘트가 필요하다는 것 자체가 위젯 부적합 신호(생년월일엔 연/월/일 드롭다운이 적합). [효율성]
- **P2 · 생년월일·Comments 등 가입 마찰 큼.** 학습 서비스 가입에 코멘트 필드까지. [전환]

### A-8. UserEditView (회원정보 수정)

- **P1 · SignUpView와 거의 동일한 폼이 복제됨** — 카피·검증·다이얼로그 중복. [일관성·유지보수]
- **P2 · 회원 탈퇴 트리거가 평범한 텍스트 "탈퇴 하시겠습니까?"** 본문 하단에 그냥 텍스트로. 위험 동작인데 시각적 구분 약함(파괴적 동작 강조 부족). [에러 예방·시각 위계]
- **P2 · 제목이 `[ My Page ]` 대괄호 표기.** [타이포·톤]

### A-9. OauthLogin / ErrorView

- **P2 · ErrorView 카피 영문("Error Occured" — 오타, "Requested resource is not available").** 'Occured'는 'Occurred' 오타, 그리고 학생 친화적이지 않음. [완성도·인지 일치]
- **P1 · ErrorView가 import 주석 처리된 `<AppConfig simple />`를 렌더.** 렌더 시 미등록 컴포넌트 경고/실패 가능. [구현 위생 → 02번]
- **P2 · OauthLogin(24줄)은 콜백 처리용으로 UI 거의 없음** — 로딩 인디케이터 없어 "멈춘 듯"한 순간 발생 가능. [로딩 상태]

### A-공통. 횡단 이슈

- **P1 · 영문/한글·대시·원문자 혼용**이 전 화면에 퍼져 있음(제목·토스트 "Confirmed"·컬럼 헤더). [일관성]
- **P1 · 컬러 토큰 부재.** 인디고 테마 기본값 + Cytoscape 하드코딩 색(`#6466f1`, `#ff6348`, `#1e90ff`, `yellow`…) + 카피별 빨강/파랑 인라인. 브랜드 컬러 한 벌이 없음. [시각 위계·일관성]
- **P2 · 빈 상태/로딩/에러 패턴이 화면마다 제각각**(빈 표 / console.error만 / 토스트 / 다이얼로그). [일관성]
- **P2 · 모든 데이터 fetch 실패가 `console.error`로만 처리** → 사용자에겐 무반응. [에러 가시성]

## 3. B. 리디자인 방향 (비주얼/UX 컨셉)

"전체적으로 갈아엎고 싶다"를 받아, 방향 2개를 제시한다. 둘 다 **관리자 사이드바를 버리고 학습 여정 중심 셸**로 가는 점은 공통이며, 톤이 다르다.

### 방향 1 — "차분한 학습 노트" (권장)
- **레퍼런스 톤:** Khan Academy의 신뢰감 + Notion/Linear의 절제된 여백 + 토스(toss)식 한국어 카피 친절함.
- **컬러:** 인디고 단일 프라이머리를 유지하되 **채도를 낮춘 1 브랜드 컬러 + 중립 그레이 스케일 + 의미색(성공/주의/위험) 3색**으로 축소. 그래프 학년 색은 **12색 → 학교급 3색(초/중/고) + 명도 단계**로 단순화하고 흰 배경 대비를 보장(노란색 퇴출).
- **타이포 스케일:** 본문 16px 기준 1.25 모듈러(16/20/25/31/39). 현재처럼 text-2xl/4xl을 라벨에 남발하지 않음. 한글 가독 글꼴(Pretendard 류) 1종.

- **셸 구조:** 사이드바 폐기 → **상단 가벼운 글로벌 내비(로고 / 개념탐색 / 진단 / 내 기록 / 로그인)** + 학습 여정은 **상단 스텝 인디케이터(진단→채점→분석→맞춤)**로. 모바일은 하단 탭.

### 방향 2 — "친근한 에듀테크" (대안)
- **레퍼런스 톤:** 산리오/콴다(Qanda)식 밝고 둥근, 일러스트·캐릭터 활용, 초·중등 친화.

- 리스크: 1인 개발에서 일러스트·캐릭터 자산 유지비가 큼. 신뢰(학부모 결제)엔 방향 1이 유리.

→ **권장: 방향 1.** 1인 개발 유지보수성·신뢰(학부모)·기존 인디고 자산 재활용에서 우위. 방향 2 요소(진척 표시)는 방향 1 위에 부분 차용.

## 4. C. 디자인 시스템 권장안 (비교표 + 권장 1안)

> 02번(React 이주)이 어떻게 결론 나든 **양쪽에서 성립**하는 권장을 지향. 핵심 통찰: 지금 문제는 "어떤 라이브러리냐"보다 **(1) 룩앤필을 라이브러리 기본 테마에 위임했다는 점**과 **(2) 디자인 토큰(색·간격·타이포)이 한 벌로 없다는 점**이다. 따라서 권장의 1순위는 **토큰 레이어 도입**이고, 컴포넌트 라이브러리는 그 위 선택지다.

| 평가 축 | Prime 계열 유지(PrimeVue→PrimeReact) | **Tailwind + 헤드리스(shadcn/Radix 계열)** | MUI / 기타 성숙 라이브러리 |
|---|---|---|---|
| 룩앤필 차별화 | 낮음 (템플릿 인상 잔존) | **높음** (디자인부터 직접) | 중 (MUI 특유 룩 고착) |
| 구축·유지 비용 | **낮음** (마이그레이션 최소) | 중~높음 (초기 구성·접근성 직접) | 중 |
| 접근성 기본기 | 중 (컴포넌트별 편차) | **높음** (Radix 등 a11y 우선) | 높음 |
| React 이주 궁합 | 높음 (PrimeReact 1:1) | **높음** (프레임워크 무관) | 높음 |
| 1인 개발 유지보수성 | **높음** (복합위젯 기성품) | 중 (직접 구성분 늘어남) | 중~높음 |
| 복합 위젯(DataTable/Tree/Calendar) | **강함** | 약함 (직접/추가 라이브러리) | 강함 |
| 그래프(Cytoscape) 궁합 | 무관 | 무관 | 무관 |

### 권장: **하이브리드 — "Tailwind + 디자인 토큰" 레이어를 도입하고, 복합 위젯은 컴포넌트 라이브러리(현 PrimeVue / 이주 시 PrimeReact)를 커스텀 프리셋으로 스킨"**

근거:
1. **MMT의 약점은 폼·레이아웃·카피·컬러(직접 만들기 쉬운 영역)에 집중**돼 있고, 강점이 필요한 건 DataTable·Tree·Calendar 같은 **복합 위젯(직접 만들면 비싼 영역)**이다. → 전자는 Tailwind+헤드리스로 차별화하고, 후자는 Prime을 **unstyled/pass-through로 토큰 주입**해 룩앤필만 통일하는 게 1인 개발에 최적.
2. **프레임워크 결정과 분리된다.** Tailwind·토큰·헤드리스 패턴은 Vue/React 양쪽 동일. 02번이 React로 가면 PrimeReact로 1:1 교체하되 같은 토큰을 그대로 씀.
3. **"순수 shadcn 전면 채택(복합위젯 직접 구현)"은 비권장** — DataTable/Tree를 직접 만드는 비용이 1인 개발엔 과하다.

실행 형태(요약): `tokens.css`(색/간격/타이포 CSS 변수) → Tailwind `theme.extend`가 토큰 참조 → Prime 테마를 토큰 기반 프리셋으로 덮어쓰기(또는 PrimeVue unstyled + pass-through). **최종 결정은 사용자가 한다.**

## 6. 사용자가 결정해야 할 열린 질문

1. **주 타깃이 누구인가?** 학생 본인(동기·게이미피케이션) vs 학부모(신뢰·결제) — 방향 1/2 및 컬러·톤 결정의 분기점.
2. **셸을 정말 갈아엎을 것인가?** 사이드바 폐기(권장)는 인상 개선 폭이 가장 크지만 라우팅/레이아웃 손이 가장 많이 간다. P1으로 두되 의사결정 필요.
3. **디자인 시스템 방향(C):** 하이브리드(권장) vs Prime 유지 vs 전면 Tailwind/shadcn — 02번 React 이주 결론과 함께 확정.
4. **ResultView에 차트를 새로 도입할 것인가?** 현재 차트 없음. 요약 게이지·정답률 도넛 정도는 효과 대비 저비용(이미 chart.js 의존성 존재). 도입 여부 결정.
5. **그래프 학년 색 체계:** 현행 12색(교육과정 충실) vs 학교급 3색+명도(가독·접근성) — 교육적 정합성과 가독성의 트레이드오프.
6. **가입 마찰:** 생년월일·Comments 등 필수 입력을 유지할지(데이터 가치 vs 전환율).
7. **"진단 학습지 저작권 고지"의 위치·톤:** 필요한 고지를 어디서 어떻게 보여줄지(상단 빨강 → 톤 다운 위치 조정).
